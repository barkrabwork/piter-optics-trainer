# Питер Оптика — Тренажёр менеджеров

## Описание проекта
Веб-тренажёр для обучения менеджеров оптики навыкам продаж по телефону. Реализован как одиночный HTML-файл без сборки. Данные хранятся в Google Таблице, бэкенд — Google Apps Script.

---

## Структура проекта

```
optics_trainer/
├── optics_trainer.html   — единственный файл проекта (SPA)
└── CLAUDE.md             — документация
```

Код Apps Script **встроен в HTML** (функция `generateScriptCode()`) и отображается в настроечном модале для копирования в Google Apps Script.

---

## Архитектура

### Фронтенд
- Чистый HTML/CSS/JS без зависимостей и сборки.
- Состояние хранится в:
  - `localStorage`: URL скрипта (`STORAGE_KEY`), ID таблицы (`SHEET_ID_KEY`).
  - `sessionStorage`: сессия пользователя (`po_session`, живёт 8 часов).
  - JS-переменные: `app` (URL, sheetId, session), `state` (активный звонок).
- Все запросы к бэкенду — через `api(params)` → `fetch(scriptUrl + query)` → JSON.

### Бэкенд (Google Apps Script)
- Единственный endpoint: `doGet(e)` — GET-запрос с параметром `action`.
- Авторизация через токен (UUID), хранится в листе «Сессии».
- Токен передаётся в каждом запросе как `?token=...`.

| `renderChecklist()` | Рендерит левую панель чек-листа |

| `api(params)` | Универсальная обёртка для GET-запросов к Apps Script |

## Статические данные (встроены в HTML)

- `STEPS` — 6 шагов чек-листа звонка (приветствие → завершение)
- `PRODUCTS` — резервный каталог товаров (используется если лист «Товары» недоступен)
- `PRODUCTS_DATA` — рабочий массив товаров (инициализируется из `PRODUCTS`, заменяется данными из таблицы)
- `FAQS` — 7 вопрос/ответ для справочника
- `PROCEDURES` — 3 процедуры (перемещение товара, жалоба, запись)

## Договорённости по разработке

- **Один файл**: весь код — в `optics_trainer.html`. Не разбивать на несколько файлов.
- **Нет зависимостей**: никаких npm, webpack, библиотек. Только ванильный JS.
- **Обратная совместимость**: изменения в формате данных не должны ломать существующие листы.
- **Graceful degradation**: при отсутствии листа «Товары» — показывать статические данные или пустой список, но не бросать ошибку.
- **Apps Script код**: всегда обновлять `generateScriptCode()` при добавлении новых API-методов.
- **Комментарии**: на русском или английском, только там, где логика неочевидна.
- **CSS переменные**: использовать только из `:root` (--primary, --success, --danger и т.д.)

## Часто задаваемые вопросы

**Q: Где хранится код Apps Script?**
В функции `generateScriptCode()` в HTML-файле. Пользователь копирует его оттуда в редактор Apps Script.

**Q: Как добавить новое API-действие?**
1. Добавить `case 'myAction': return r(myAction(p));` в `doGet` внутри `generateScriptCode()`.
2. Добавить функцию `myAction(p)` в строку-шаблон.
3. При необходимости — добавить вызов в нужном месте фронтенда.
4. Пользователь должен скопировать обновлённый скрипт и перевыпустить деплой.

**Q: Как авторизация работает при первом запуске?**
Пользователь «Начальник» создаётся автоматически (функция `uSheet()`) с пустым паролем. При первом входе с пустым паролем — `first_login` flow → экран установки пароля.

**Q: Что происходит при ошибке загрузки товаров?**
`loadProductsFromSheet()` при ошибке оставляет `PRODUCTS_DATA` без изменений (статический каталог). Ошибка логируется в консоль, пользователю не показывается.

**Q: Почему checkProductsSheet вызывается каждый раз при открытии дашборда?**
Начальник должен знать, что лист «Товары» не создан, до того как менеджеры начнут работать. Если лист отсутствует — дашборд заменяется страницей с кнопкой «Создать лист».

## Инструкция по первому запуску

> Для человека, который первый раз разворачивает систему. Следуйте пунктам по порядку.

---

### Часть 1 — Создание Google Таблицы

1. Откройте [Google Sheets](https://sheets.google.com) и создайте **новую пустую таблицу**.
2. Дайте ей понятное название, например: `Питер Оптика — Тренажёр`.
3. Скопируйте **ID таблицы** из адресной строки браузера — это длинная строка между `/d/` и `/edit`:
   ```
   https://docs.google.com/spreadsheets/d/ВОТ_ЭТОТ_ID/edit
   ```
   Сохраните его — понадобится на следующем шаге.

> **Права доступа к таблице:** делиться таблицей ни с кем не нужно. Скрипт обращается к ней от имени владельца автоматически. Менеджеры работают только через веб-интерфейс.

---

### Часть 2 — Настройка Apps Script

**2.1. Открытие редактора скрипта**

1. В Google Таблице нажмите меню **Расширения → Apps Script**.
2. Откроется редактор скрипта в новой вкладке.

**2.2. Получение кода скрипта**

1. Откройте файл `index.html` в браузере (двойной клик по файлу).
2. На экране входа нажмите ссылку **«⚙ Настройка скрипта Google Apps»**.
3. В открывшемся окне в поле **«ID таблицы»** вставьте ID из шага 1.3.
4. Нажмите кнопку **«Копировать»** над блоком с кодом.

**2.3. Вставка кода**

1. В редакторе Apps Script удалите весь существующий код (Ctrl+A → Delete).
2. Вставьте скопированный код (Ctrl+V).
3. Нажмите **Ctrl+S** (или кнопку 💾 «Сохранить проект»).

**2.4. Первое развёртывание**

1. Нажмите кнопку **«Развернуть»** (синяя, правый верхний угол) → **«Новое развёртывание»**.
2. Нажмите на шестерёнку ⚙ рядом с «Тип» → выберите **«Веб-приложение»**.
3. Заполните поля:
   - **Описание:** можно оставить пустым или написать «v1»
   - **Запускать как:** «Я» (ваш аккаунт)
   - **Кто имеет доступ:** выбрать **«Все»** ⚠️ (не «Все, у кого есть аккаунт Google» — иначе запросы из приложения не пройдут)
4. Нажмите **«Развернуть»**.

**2.5. ⚠️ Разрешение для Google (обязательный шаг)**

После нажатия «Развернуть» Google покажет экран «Этому приложению не предоставлен доступ» или «Это приложение не проверено». Это нормально — скрипт не публичный, и Google предупреждает об этом.

Действия:
1. Нажмите **«Дополнительные настройки»** (или «Advanced») — небольшая серая ссылка внизу.
2. Появится ссылка **«Перейти на страницу [название проекта] (небезопасно)»** — нажмите её.
3. Проверьте список запрашиваемых разрешений (Таблицы, Gmail) и нажмите **«Разрешить»**.

**2.6. Копирование URL скрипта**

1. После успешного развёртывания появится окно с **«URL веб-приложения»**.
2. Скопируйте его — строка вида:
   ```
   https://script.google.com/macros/s/AKfyc.../exec
   ```
3. Вернитесь в `index.html` → окно настройки → вставьте URL в поле **«URL веб-приложения»** → нажмите **«Сохранить»**.

### Часть 3 — Инициализация системы

**3.1. Первый вход**

1. Закройте окно настройки.
2. В поле **«Имя»** введите: `Начальник`
3. Поле **«Пароль»** оставьте **пустым** и нажмите **«Войти»**.
4. Система предложит установить пароль — придумайте пароль для начальника и введите его дважды.
5. После этого откроется дашборд начальника.

**3.2. Создание листа «Товары»**

1. Если на дашборде появилась кнопка **«📋 Создать лист «Товары»»** — нажмите её.
2. Дождитесь уведомления «Лист создан».

**3.3. ⚠️ Авторизация Gmail для отправки обратной связи (обязательный шаг)**

Форма «Написать нам» (кнопка в интерфейсе) отправляет письма через Gmail. Чтобы это заработало, нужно один раз вручную запустить функцию авторизации:

1. Вернитесь во вкладку редактора Apps Script.
2. В выпадающем списке функций (рядом с кнопкой ▶) выберите функцию **`triggerMailAuth`**.
3. Нажмите ▶ **«Выполнить»**.
4. Google снова покажет экран разрешений — действия те же, что в пункте 2.5 (нажать «Advanced» → разрешить).
5. В журнале выполнения внизу появится: `GmailApp авторизован.` — это значит, всё сработало.

**3.4. Добавление менеджеров**

1. В дашборде начальника нажмите кнопку **«+ Добавить сотрудника»**.
2. Введите имя менеджера (как он будет входить в систему).
3. Повторите для каждого менеджера.
4. Каждый менеджер при первом входе введёт своё имя, оставит пароль пустым и сам установит пароль.

### Часть 5 — При обновлении скрипта

Если в коде были изменения (новые функции, исправления):

1. Откройте Apps Script → вставьте новый код → сохраните.
2. **«Развернуть»** → **«Управление развёртываниями»** → нажмите ✏ **«Изменить»** у активного развёртывания.
3. В поле «Версия» выберите **«Новая версия»** → нажмите **«Обновить»**.
4. URL скрипта **не изменится** — в настройках приложения ничего менять не нужно.
